# 自适应并发与部署打包设计补充

- 日期：2026-04-24
- 主题：为现有视频打标工具增加自适应并发处理策略、改进画面描述约束，并输出可部署目录与 README
- 状态：待用户审阅

## 1. 背景

当前工具已经支持：

- 本地视频扫描
- 视频压缩
- 调用 Qwen/DashScope 分析视频
- 输出结构化单选字段与多选字段
- 输出 review 审核清单
- 使用配置文件控制主要参数

用户进一步提出两个实际需求：

1. **效率问题**：
   - 需要提升批处理速度
   - 希望根据 DashScope/Qwen 的并发能力设计合理的并发方案
   - 如果模型调用支持并发，则应支持并发分析
   - 如果模型调用不适合高并发，则至少要把压缩阶段并发化，再串行或低并发地进行模型分析

2. **部署问题**：
   - 需要把项目整理为一个可直接部署到其他电脑上的目录
   - 需要 README 说明如何安装、配置、运行
   - 所有环境相关参数都应该通过配置文件调整，而不是改代码

## 2. 目标

1. 引入自适应并发处理策略。
2. 视频压缩支持并发。
3. 模型请求支持小并发起步，并在失败时自动退避。
4. 并发参数可配置。
5. 继续强化 `画面描述` 中对“手机时间特写开场”的忽略规则。
6. 产出一个清晰的部署目录和 README，支持在其他电脑快速部署。

## 3. 推荐方案

采用“**压缩并发 + 模型小并发 + 退避重试 + 全配置化**”方案。

### 3.1 压缩阶段

- 压缩多个视频时并发执行
- 由配置控制压缩并发数
- 适合使用线程池或进程池调度 `ffmpeg`

### 3.2 模型阶段

- 模型分析不采用激进高并发
- 默认使用小并发（例如 2）
- 并发数由配置控制
- 出现网络错误、限流或超时时自动指数退避

### 3.3 错误恢复

- 单条视频失败不影响整批
- 保留已完成的压缩产物和中间 JSON
- 失败项记录到日志或 review 结果中，便于后续重跑

### 3.4 部署交付

产出一个明确的部署目录，例如：

- `deployment_package/`

目录中包含：

- 运行代码
- 配置模板
- README
- 依赖说明
- 可选示例输出目录结构说明

## 4. 并发策略细化

### 4.1 压缩并发

配置项建议：

- `compression_workers`

行为：

- 根据视频数和配置值调度压缩任务
- 每个视频独立生成压缩产物
- 压缩阶段完成后，再进入分析阶段

### 4.2 模型并发

配置项建议：

- `provider_workers`
- `max_retries`
- `retry_backoff_seconds`
- `retry_backoff_multiplier`

行为：

- 初始小并发请求 Qwen
- 遇到 429、连接错误、超时等错误时退避重试
- 避免瞬间将全部视频同时打给模型

### 4.3 自适应退避

- 首次失败后等待短时间
- 后续失败指数增加等待时间
- 到达最大重试次数后标记该任务失败

## 5. 配置文件扩展

现有配置应继续扩展，建议增加：

```json
{
  "concurrency": {
    "compression_workers": 2,
    "provider_workers": 2,
    "max_retries": 3,
    "retry_backoff_seconds": 2,
    "retry_backoff_multiplier": 2
  }
}
```

结合此前的路径配置、provider 配置、prompt 配置和标签模板配置，形成完整可迁移的配置体系。

## 6. 画面描述约束强化

用户已经指出：

- 当前 `画面描述` 仍可能描述“视频起始于手持手机展示时间的特写”
- 这一段属于固定无效开场，不应进入描述

因此新的 prompt 规则需要更强：

- 不只是“尽量忽略”
- 而是明确要求：
  - 不得把该固定开场当成有效内容写入 `画面描述`
  - 画面描述应从真正进入测试场景之后开始描述

## 7. 部署目录建议

建议生成如下部署目录：

```text
deployment_package/
  video_tagging_assistant/
  tests/                # 可选，若希望保留验证能力
  videos/               # 空目录或示例说明
  output/               # 空目录或示例说明
  default_config.json
  README.md
  qwen_video_compress_and_test.py   # 可选保留为手工验证脚本
  pytest.ini             # 若保留测试
```

也可以采用更轻量版本，只保留运行所需部分。

## 8. README 内容建议

README 至少应包含：

1. 项目作用说明
2. 环境要求
   - Python 版本
   - `ffmpeg.exe` 要求
   - 依赖安装方式
3. 配置文件说明
4. 如何准备视频目录
5. 如何运行主 CLI
6. 如何查看 review 输出
7. 如何修改标签候选项
8. 如何迁移到其他电脑
9. 安全提示（特别是明文 API Key 风险）

## 9. 风险与约束

1. DashScope 的实际限流配额与账号、地域、模型有关，不能假设可无限并发。
2. 并发过高可能导致 429、超时或连接中断。
3. 压缩并发过高会占用本地 CPU，影响机器响应速度。
4. 明文 API Key 放入配置文件方便部署，但存在泄露风险。

## 10. 验收标准

满足以下条件即可认为这一轮扩展可用：

1. 压缩阶段支持并发。
2. 模型阶段支持小并发与退避重试。
3. 并发参数可在配置文件中修改。
4. `画面描述` 不再写固定手机时间开场。
5. 工具可整理成独立部署目录。
6. README 足够让用户在另一台电脑完成部署和运行。
