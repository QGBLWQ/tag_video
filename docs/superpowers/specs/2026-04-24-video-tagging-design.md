# 视频简介与标签自动化辅助流程设计

- 日期：2026-04-24
- 主题：将现有分散脚本整合为一套离线批处理工具，用于压缩视频、调用低成本模型生成简介与标签，并输出本地审核清单
- 状态：已确认设计，待用户审阅书面 spec

## 1. 背景

当前工作目录中已有若干围绕 ADB 拉取、目录整理、文件计数的脚本，例如 `pull.py`、`check.py`、`count_rk.py` 以及配套的 `.bat` 文件。已有流程说明还体现在 `PC_A_采集记录表v2.1.xlsm` 中：采集数据按目录组织，人工需要根据视频内容为每条视频撰写简介并打标签，再自行填写到表格中。

当前痛点不是“把 Excel 自动填完”，而是“人工从零写简介和标签”这一步过于繁琐，且重复劳动较多。

本设计的目标是：在保留人工审核和人工填表的前提下，新增一条自动化辅助链路，把视频压缩后送给低成本模型生成候选简介与标签，再输出为本地审核清单，供人工确认后手动填写到正式表格。

## 2. 目标与非目标

### 2.1 目标

1. 将现有零散流程整合为一个统一的本地离线批处理工具。
2. 支持从本地视频中批量生成“建议简介”和“建议标签”。
3. 在送模前先生成低成本代理视频，降低模型调用成本。
4. 组装结构化上下文，包含：
   - 视频本身
   - 目录/文件名提取的信息
   - 来自现有记录规范的固定模板信息
5. 模型接入层支持多供应商切换，通过统一接口隐藏差异。
6. 将结果输出为本地文本审核清单，便于人工审核、修改和确认。
7. 不影响现有 ADB 拉取与目录管理脚本的继续使用。

### 2.2 非目标

1. 不自动修改 `PC_A_采集记录表v2.1.xlsm`。
2. 不自动替代人工最终审核。
3. 不在第一版中提供本地网页 UI。
4. 不尝试重构与本任务无关的历史脚本。
5. 不追求一次性支持所有可能的视频来源和所有表格结构。

## 3. 设计原则

1. **最小闭环优先**：第一版只完成“批量生成候选结果 + 本地审核”的核心价值。
2. **低成本优先**：通过压缩代理视频和可切换模型供应商控制调用成本。
3. **人工可控优先**：所有结果都先进入审核清单，不直接写 Excel。
4. **结构清晰优先**：将扫描、压缩、上下文构建、模型调用、结果导出拆成独立模块。
5. **可重跑优先**：中间产物落地，单条失败可重试，不要求整批重跑。

## 4. 推荐方案

在已讨论的三个方案中，最终采用：

**方案 A：离线批处理流水线**

流程如下：

1. 扫描待处理视频
2. 生成送模用压缩视频
3. 从目录名、文件名、固定模板中构建结构化上下文
4. 调用可配置的低成本模型生成简介与标签
5. 输出本地文本审核清单

选择该方案的原因：

- 与现有工作方式最贴近
- 不引入 UI 复杂度
- 更容易稳定落地
- 后续若需要网页审核界面，可在此基础上增量升级

## 5. 系统边界

该工具位于现有数据整理流程之后、Excel 填写之前。

### 输入

- 本地原始视频文件或待处理视频目录
- 视频所在目录及文件名中的结构化信息
- 固定模板信息（例如记录规范、常见标签维度、已有字段提示）
- 模型供应商配置

### 输出

- 压缩后的代理视频文件
- 每条视频的结构化中间结果
- 汇总审核清单（本地文本文件）
- 执行日志

### 不直接处理的内容

- Excel 自动写入
- 最终发布或提交动作
- 复杂的多人协同审核

## 6. 核心架构

系统拆分为 5 个核心组件和 1 个流程协调层。

### 6.1 scanner

职责：

- 扫描指定目录下待处理视频
- 识别视频文件列表
- 提取基础元信息，如文件名、父目录、相对路径
- 为后续步骤创建任务单元

输出：视频任务列表。

### 6.2 compressor

职责：

- 将原视频压缩为适合送模的低成本代理视频
- 控制分辨率、码率、时长或抽帧策略
- 将压缩结果与源视频建立映射

输出：代理视频路径及压缩元数据。

### 6.3 context_builder

职责：

- 从目录名、文件名中提取 case、模式、设备等可用上下文
- 结合固定模板，形成统一的结构化提示输入
- 在信息不足时做降级处理并标记风险

输出：标准化模型输入上下文。

### 6.4 llm_client

职责：

- 提供统一的模型调用接口
- 屏蔽不同供应商在认证、请求格式、响应格式上的差异
- 将模型返回内容清洗为固定结果结构

输出：标准化结果对象，例如：

- 建议简介
- 建议标签列表
- 备注
- 原始响应摘要

### 6.5 review_exporter

职责：

- 将结构化结果导出为本地文本审核清单
- 提供便于人工查看和复制的格式
- 保留源路径与状态字段，支持人工追踪

输出：审核清单文件。

### 6.6 orchestrator

职责：

- 串联以上所有步骤
- 管理任务状态
- 控制重试、跳过、失败记录和日志输出

输出：整批任务执行结果。

## 7. 数据模型

建议以“每个视频一个任务单元”为核心。

### 7.1 VideoTask

字段建议包括：

- `source_video_path`
- `relative_path`
- `file_name`
- `case_id`（可选）
- `mode`（可选）
- `device_info`（可选）
- `status`

### 7.2 CompressedArtifact

字段建议包括：

- `source_video_path`
- `compressed_video_path`
- `duration`
- `resolution`
- `size_bytes`
- `compression_profile`

### 7.3 PromptContext

字段建议包括：

- `source_video_path`
- `compressed_video_path`

### 7.4 GenerationResult

## 8. 端到端数据流

1. 用户指定待处理目录。
2. `scanner` 发现视频文件并生成任务列表。
3. `compressor` 为每个任务生成代理视频。
4. `context_builder` 解析目录/文件名，并合并固定模板，生成送模上下文。
5. `llm_client` 根据统一接口调用默认供应商或用户配置的供应商。
6. 响应被清洗为固定结构：简介、标签、备注。
7. `review_exporter` 将所有任务导出为本地审核清单。
8. 用户人工审核文本清单后，自行填写正式 Excel。

## 9. 审核清单格式

第一版采用本地文本清单，不引入网页界面。

建议每条记录至少包含以下字段：

- 视频名
- 源文件路径
- case 或目录标识
- 建议简介
- 建议标签
- 备注
- 审核状态（默认未审核）

文本格式要求：

- 便于人工阅读
- 可逐条复制
- 保留分隔标记，方便后续如果需要再转换成其他格式

## 10. 错误处理

### 10.1 压缩失败

- 当前任务标记失败
- 记录错误原因
- 不阻塞其他视频继续处理

### 10.2 模型调用失败

- 保留压缩产物和上下文
- 支持单条重试
- 不强制整批重跑

### 10.3 上下文不足

- 当目录名/文件名无法提取足够信息时，降级为“视频 + 固定模板”模式
- 在结果中明确标注“上下文不足”

### 10.4 响应格式异常

- 适配层负责清洗和容错
- 若解析失败，记录原始片段并标记该任务待人工补充

### 10.5 日志要求

至少区分以下类别：

- 压缩失败
- 上传或调用失败
- 模型超时
- 响应解析失败
- 人工待补充

## 11. 配置设计

需要一个本地配置文件，统一管理以下内容：

- 待处理目录
- 压缩参数
- 默认模型供应商
- 可选供应商列表
- API 密钥来源
- 提示模板
- 导出目录
- 日志目录

配置应支持：

- 切换默认供应商
- 追加新供应商适配器
- 修改输出路径而不改代码

## 12. 供应商适配策略

模型调用层采用统一接口，供应商只是实现细节。

统一接口至少需要支持：

- 传入视频与结构化上下文
- 返回建议简介、标签、备注
- 统一错误对象

这允许第一版先接入一个默认供应商，后续再扩展其他低成本模型，而不影响主流程。

## 13. 与现有脚本的关系

现有 `pull.py`、`check.py`、`count_rk.py` 主要负责 ADB 拉取、核对、统计。新工具不替代这些脚本的职责，而是补在“视频已整理到本地之后”的环节。

也就是说：

- 现有脚本仍可继续使用
- 新工具只关注视频理解与候选文案生成
- 后续若需要，可以再考虑与已有目录结构更深度集成，但不作为第一版前提

## 14. 测试策略

### 14.1 小样本流程验证

先选取 3–5 个典型视频，跑通完整流程，确认：

- 压缩后视频仍保留足够语义
- 模型生成的简介和标签具备参考价值
- 审核清单适合人工使用

### 14.2 单元测试重点

对以下部分补测试：

- 目录/文件名元信息提取
- 模板拼装逻辑
- 标签结果清洗与规范化
- 供应商响应解析

### 14.3 集成测试

进行一次小规模端到端测试，不追求大规模批量性能，只验证闭环正确性。

### 14.4 验收标准

验收以人工效率提升为核心，标准包括：

- 人工不再需要从零编写简介和标签
- 大多数视频只需少量修改即可使用生成结果
- 审核过程不需要频繁切换额外工具

## 15. 后续可扩展方向

这些不是第一版目标，但设计应为其保留余地：

1. 本地网页审核界面
2. 导出为更适合复制进 Excel 的结构化格式
3. 按目录规则自动套用不同提示模板
4. 对单个视频进行增量重跑
5. 接入更多模型供应商

## 16. 实施范围结论

本项目第一版聚焦于一个明确且受控的闭环：

- 输入本地视频
- 压缩并调用低成本模型
- 结合目录信息与固定模板生成建议简介和标签
- 导出本地文本审核清单
- 人工审核后自行填写正式 Excel

这是一个“自动化辅助系统”，不是“全自动填表系统”。它的核心目标是显著减少人工重复劳动，同时保留最终质量控制权。
