# 基于 tag_pic 的结构化单选标签输出设计补充

- 日期：2026-04-24
- 主题：将现有 Qwen 视频理解输出从自由 `summary/tags/notes` 改为基于 `tag_pic.png` 的结构化单选字段输出
- 状态：待用户审阅

## 1. 变更背景

当前系统已经完成 Qwen/DashScope 视频理解链路接入，并能从压缩视频中得到中文简介、标签和备注。但这套输出仍偏向“自由描述”，不适合当前运动相机测试数据打标流程。

用户补充说明：本项目实际需要的标签不是自由生成的标签数组，而是严格遵循本地 `tag_pic.png` 所体现的标签体系。典型字段包括：

- 安装方式
- 运动模式
- 运镜方式
- 光源
- 画面描述（备注性描述）

其中前四类属于**单选字段**，模型只能从每个字段的候选值中选一个；`画面描述`用于描述亮度变化、场景细节、画面变化等补充信息。

第一版目标采用用户确认的 **方案 C**：

- 只保留结构化字段输出
- 暂不保留自由 `tags` 数组

## 2. 目标

1. 将模型输出从自由标签改为结构化字段。
2. 单选标签字段必须只返回一个值。
3. 单选字段的值必须来自 `tag_pic.png` 对应的标签体系。
4. 增加 `画面描述` 字段，用于表达场景细节、亮度变化、运动特征等。
5. 审核清单和中间 JSON 产物都应体现新的结构化输出。

## 3. 非目标

1. 本轮不恢复自由 `tags` 数组。
2. 本轮不做多标签混合输出。
3. 本轮不做 UI 级的标签编辑器。
4. 本轮不尝试自动识别 Excel 中的字段并自动回填。

## 4. 推荐实现方式

采用“**人工固化标签模板 + prompt 强约束**”的方式：

- 人工根据 `tag_pic.png` 抽取标签字段和每个字段允许的候选值
- 将这些字段定义固化进配置或模板中
- provider 在构造 prompt 时明确要求模型：
  - 每个字段必须返回且只能返回一个值
  - 值必须从给定候选列表中选择
  - 不允许输出额外标签字段
  - `画面描述` 使用自然语言补充说明

选择该方式的原因：

- 最小改动即可落地
- 与现有 Qwen provider 兼容性最好
- 输出稳定性高于自由标签方式
- 比“从图片自动解析标签体系”更可靠

## 5. 输出结构

建议模型输出改为以下 JSON 结构：

```json
{
  "安装方式": "候选值之一",
  "运动模式": "候选值之一",
  "运镜方式": "候选值之一",
  "光源": "候选值之一",
  "画面描述": "描述亮度变化、场景细节、运动特征等"
}
```

要求：

- 单选字段必须严格一值
- 不允许返回数组
- 不允许输出未定义字段
- 如果视频信息不足，也必须在候选值中选择最合理项，并在 `画面描述` 中说明不确定性

## 6. 系统改动点

### 6.1 prompt_template

现有 `prompt_template` 从“自由摘要 + 标签规则”调整为“结构化字段模板”。

新增内容：

- 标签字段名
- 每个字段的候选值列表
- 输出 schema
- 单选约束
- `画面描述` 的书写要求

### 6.2 Qwen provider

provider 的 prompt 生成逻辑要改成：

- 将视频内容、目录元数据、字段候选值一起发给模型
- 让模型只输出结构化 JSON
- 不再要求 `summary/tags/notes`

同时保留：

- fenced JSON 清洗逻辑
- JSON 解析逻辑

### 6.3 数据模型

当前 `GenerationResult` 主要偏向：

- `summary_text`
- `tags`
- `notes`

需要扩展或替换为结构化字段承载方式，保证：

- 中间 JSON 能存储结构化标签
- 审核清单能展示每个标签字段
- 后续如果恢复 `tags` 数组，也有扩展空间

### 6.4 审核清单

审核清单应改成更适合结构化核验的形式，至少包含：

- 视频路径
- 安装方式
- 运动模式
- 运镜方式
- 光源
- 画面描述
- 审核状态

## 7. 风险与约束

1. 如果 `tag_pic.png` 中的候选值很多，prompt 会变长，增加 token 消耗。
2. 如果候选值定义不清晰，模型会在相近选项间摇摆。
3. 如果某些字段其实应该允许多选，但当前按单选强约束，可能损失表达准确性。
4. 当前方案依赖人工把 `tag_pic.png` 转成结构化模板，因此第一步需要人工确认字段和候选值。

## 8. 推荐推进顺序

1. 先人工确认 `tag_pic.png` 的字段和每个字段的候选值。
2. 将这些候选值写入配置。
3. 修改 provider prompt。
4. 修改中间结果与审核清单格式。
5. 补测试并重新跑端到端验证。

## 9. 验收标准

满足以下条件即可认为第一版可用：

1. 模型稳定输出固定字段 JSON。
2. 单选字段不再自由发散。
3. 审核清单可直接用于人工核对。
4. `画面描述` 能反映亮度变化、画面细节、场景变化。
5. 对同类视频重复测试时，字段输出具备较高稳定性。
