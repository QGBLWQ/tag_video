# 多选标签与全量配置化设计补充

- 日期：2026-04-24
- 主题：在现有结构化打标基础上，新增多选字段、画面描述约束，并将环境路径与标签模板进一步配置化
- 状态：待用户审阅

## 1. 变更背景

当前系统已经支持：

- 使用 Qwen/DashScope 对视频进行理解
- 输出结构化单选字段（如安装方式、运动模式、运镜方式、光源）
- 输出 `画面描述`
- 通过配置文件管理基本输入输出路径与 provider 设置

用户进一步补充了两类标签需求：

### 1.1 画面特征（多选）

候选项包括：

- 纹理_高低频
- 重复纹理
- 边缘特征_强弱
- 运动对焦
- 人物肤色
- 景深远近切换
- 反射与透视

### 1.2 影像表达（多选）

候选项包括：

- 风景录像
- 建筑空间
- 美食游街
- 运动跟拍
- 主题拍摄
- 赛事舞台
- 多目标分散运镜
- 交互叙事

与此前的基础字段不同，用户明确要求：

- 这些新增字段允许**多选**，给模型一定冗余空间，方便人工筛选

同时，用户补充了对 `画面描述` 的重要约束：

- 视频中固定会出现“手持手机展示时间的特写”开场
- 这一部分属于无效固定模式，**不应写入画面描述**
- `画面描述` 可以写得更详细，重点描述有效测试内容

此外，用户计划在其他电脑上部署，因此希望进一步加强配置化能力，做到后续迁移时主要修改配置，而不是改代码。

## 2. 目标

1. 保留已有基础单选字段。
2. 新增多选字段：`画面特征`、`影像表达`。
3. `画面描述` 改为更详细的补充说明字段。
4. `画面描述` 中明确排除手机时间特写开场内容。
5. 将输入目录、输出目录、review 输出路径、提示词、标签字段与候选项都进一步配置化。
6. 提高工具在不同电脑环境下的可迁移性。

## 3. 字段类型设计

### 3.1 基础字段（单选）

以下字段继续采用单选：

- 安装方式
- 运动模式
- 运镜方式
- 光源

### 3.2 扩展字段（多选）

以下字段改为多选数组：

- 画面特征
- 影像表达

### 3.3 自由描述字段

- 画面描述

## 4. 推荐输出结构

建议模型最终输出的 JSON 结构如下：

```json
{
  "安装方式": "单选值",
  "运动模式": "单选值",
  "运镜方式": "单选值",
  "光源": "单选值",
  "画面特征": ["多选值1", "多选值2"],
  "影像表达": ["多选值1", "多选值2"],
  "画面描述": "详细描述有效画面信息"
}
```

约束：

- 单选字段必须只返回一个值
- 多选字段允许返回多个值
- 多选字段的值必须来自给定候选列表
- 不允许输出未定义字段
- `画面描述` 必须排除“手机展示时间特写”这类固定开场内容

## 5. 画面描述规则

`画面描述` 的重点改为：

- 光亮变化
- 场景细节
- 主体运动
- 画面变化
- 与测试相关的有效特征

明确排除：

- 固定出现的“手持手机展示时间”开场镜头
- 对测试无意义的固定流程性内容

描述风格：

- 可以更详细
- 鼓励比之前更充分地表达画面信息
- 不必为了简短而牺牲细节

## 6. 配置化范围扩展

为了支持后续在其他电脑上部署，以下内容应全部配置化：

### 6.1 环境路径

- 视频输入目录
- 压缩视频输出目录
- 中间 JSON 输出目录
- review 文本输出目录或 review 文件完整路径
- 日志目录（如需要）

### 6.2 Provider 配置

- 模型名
- Base URL
- API Key 环境变量名
- 可选明文 API Key
- 抽帧参数（fps）

### 6.3 Prompt 配置

- system prompt
- 画面描述规则
- 忽略固定开场内容的提示
- 标签输出规则

### 6.4 标签模板配置

- 单选字段与候选值
- 多选字段与候选值
- 字段顺序

建议配置结构扩展为：

## 8. 系统改动点

### 8.1 Provider Prompt 生成

provider 的 prompt 需要区分：

- 单选字段
- 多选字段
- 画面描述要求
- 忽略固定开场内容的规则

### 8.2 数据模型

需要支持：

- 单选字段（字符串）
- 多选字段（字符串数组）
- 详细画面描述

### 8.3 审核清单

review 输出需要能清晰展示：

- 基础单选字段
- 多选字段
- 详细画面描述

### 8.4 路径读取逻辑

现有代码部分路径是从 `output_dir` 推导得出，后续应允许显式配置具体输出路径，减少跨电脑迁移时的改动量。

## 9. 风险与约束

1. 多选字段会提高输出复杂度，模型更容易在边界选项上发散。
2. 候选值变多后，prompt 更长，token 消耗增加。
3. 如果标签定义仍不够精确，模型在多选字段上会更容易“宁可多选”。
4. 明文 API Key 放在配置文件中仍有泄露风险，但符合当前用户偏好。

## 10. 验收标准

满足以下条件即可认为这一版扩展可用：

1. 基础字段保持单选稳定输出。
2. `画面特征` 和 `影像表达` 支持多选。
3. `画面描述` 更详细，且不再写手机时间特写开场。
4. 输入输出路径和标签模板可通过配置调整。
5. 在另一台电脑部署时，无需改核心代码，只需改配置。
